前言

作为一个半路出家的前端er，工作中需要经常上网搜问题找答案，很容易就进入别人的个人博客，有些博客排版美观，外观精美，我就觉得很羡慕，然后在想，要是我也有个人博客，写写自己的东西，别人也能访问到，岂不是美滋滋。虽然对于搭建博客一无所知，但是感谢万能的互联网，站在别人(巨人)的肩膀上，本博客终于搭了起来，详见VCR。

weihome的博客诞生记

Hexo

首先面临着选择博客平台的问题，比较主流的有WordPress和Hexo。WordPress功能完善，也比较复杂，自带数据库，Hexo则是一个静态博客系统，使用方便，个人觉得排版和主题也是优于WordPress。从我自己的情况看，静态博客比较适合我，而且可以集成到github，支持markdown，写文章也是so easy，至于评论可以使用第三方服务。以后有时间或许还会折腾一下wordpress，目前就先使用hexo吧。

部署博客

现在访问到的博客，是通过 hexo server 搭建在本地的一个临时服务器，只有自己通过 http://localhost:4000 才能访问到，如果你想要把自己的博客发布的互联网上，你需要利用一个服务器来部署你的博客。

Hexo 是基于静态页面的博客系统，意味着对服务器的要求可以非常的低。Github 和 Coding.net 就提供了免费的静态页面的托管服务，我们也就正好可以利用这个特性把我们的博客部署到这两个网站上面（为互联网的免费精神干杯🍻）。

创建托管仓库

前面说到hexo集成到github，就是用Github Pages来托管博客，Github Pages是 Github 用来提供给个人/组织或者项目的网页服务，只需要部署到你的 Github Repository，推送代码，便可以实时呈现。

如果之前都没用过github，赶紧创建一个github账号吧，然后你就会发现打开了新世界的大门，这些先按下不表，为了部署博客先创建一个名称为 .github.io 的仓库来托管网页。

以我的 Github 为例，我的用户名是 weihomechen，所以创建一个名为 weihomechen.github.io 的仓库，创建的仓库地址便是：https://github.com/weihomechen/weihomechen.github.io.git, 创建完后，我们暂时不用管它。

绑定域名

其实到这一步，博客已经可以在网上被访问到了，美中不足的是博客的域名稍微长了一些，比如这样的： http://wehomec.github.io
如果想自己博客的域名短一些好记一些（逼格更高一些），其实这些都不是重点，重点是，你没有域名控制的自主权，DNS 和 CDN 都没法做，一切得看 Github 的网络环境。既然到了这一步了，就把域名也绑了吧。

注册域名推荐使用godaddy，俗称狗爹，是世界上最大的域名注册商。购买域名需要花一些银子，推荐使用优惠码购买（土豪可以自动忽略这个），网上一搜一大把，下面这个就挺好用的：
http://www.51pin.cn/archives/1966

一般来讲，使用网上的优惠码第一年收费8.99美元，以后每年的收费是10.99美元，不过在网上可以搜到合适的优惠码，可以每年的收费都是8.99美元，记得多测试自行鉴别。
选好域名后就到购物车结账了，可以使用支付宝，创建账户，填完信息，网站支持中文，使用起来没什么难度。结算后，重新登录，域名已经显示在你的账户了。

兼容HTML

Markdown和HTML都是标记语言，那么他们有什么关系呢？

其实markdown只对应HTML标记的一小部分，但是使用markdown书写的文档可以直接使用html的标签，markdown是兼容html的。不过有些块状标签的使用有一定的规范，比如“div、table、p” 等标签，必须在前后加上空行与其它内容区隔开，还要求它们的开始标签与结尾标签不能用制表符或空格来缩进。请注意，在 HTML 区块标签间的 Markdown 格式语法将不会被处理。比如，你在 HTML 区块内使用 Markdown 样式的_斜体_会没有效果。HTML 的区段（行内）标签如 span等可以在 Markdown 的段落、列表或是标题里随意使用。依照个人习惯，甚至可以不用 Markdown 格式，而直接采用 HTML 标签来格式化。举例说明：如果比较喜欢 HTML 的a或img标签，可以直接使用这些标签，而不用 Markdown 提供的链接或是图像标签语法。和处在 HTML 区块标签间不同，Markdown 语法在 HTML 区段标签间是有效的。
